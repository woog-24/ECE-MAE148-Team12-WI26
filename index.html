// 01 — Overview

The Problem & Solution

DonkeyCar's built-in GPS path follower updates position only as fast as the GPS receiver outputs fixes: 10 Hz. The PID control loop runs at 50 Hz, meaning it stalls on stale position data between fixes, causing jerky steering and poor path tracking.

⚠ Without Fusion

GPS-Only (10 Hz)

Position freezes between GPS fixes. PID loop outruns data. Heading comes from GPS course only, unreliable at low speeds or when stopped.

✓ With Fusion

IMU + GPS (50 Hz)

IMU propagates position forward between GPS fixes. Kalman filter blends both sources. Smooth continuous heading from BNO08x quaternion.

// 02 — Setup

Installation

Assumes DonkeyCar is already installed with a virtual environment at ~/donkey_env.

1

Install dependencies

terminal 
source ~/donkey_env/bin/activate
pip install adafruit-circuitpython-bno08x pyubx2 pyserial utm
2

Copy the fusion part into DonkeyCar

terminal 
cp gps_imu_fused_bno08x.py \
   ~/donkey_env/lib/python3.11/site-packages/donkeycar/parts/
3

Configure GPS receiver (run once)

Sets 10 Hz multi-GNSS mode. Settings are saved to battery-backed RAM, survives power cycles.

terminal 
python set_gps_10hz.py
4

Patch manage.py

Replace the entire add_gps function in ~/projects/mycar/manage.py:

manage.py 
def add_gps(V, cfg):
    if cfg.HAVE_GPS:
        from donkeycar.parts.gps_imu_fused_bno08x import GpsImuFusedBno08x
        fusion = GpsImuFusedBno08x(
            gps_port=cfg.GPS_SERIAL,
            gps_baud=cfg.GPS_SERIAL_BAUDRATE,
            debug=cfg.GPS_DEBUG,
        )
        V.add(fusion, outputs=['pos/x', 'pos/y'], threaded=True)
        return None
    return None
5

Update myconfig.py

myconfig.py 
GPS_DEBUG = True   # set False once working
GPS_SERIAL = "/dev/ttyUSB0"
GPS_SERIAL_BAUDRATE = 38400
// 03 — How It Works

Kalman Filter Design

The filter maintains a 5-state estimate (position, velocity, and heading) in a local ENU frame anchored to a UTM origin set at first GPS fix. Output is absolute UTM coordinates, matching the frame DonkeyCar uses when recording waypoints.

50 Hz — Predict

IMU Propagation

BNO08x linear acceleration is rotated from body frame to world frame using the current yaw estimate, then used to propagate position and velocity forward each cycle.

50 Hz — Yaw Update

Heading from Quaternion

BNO08x game rotation vector gives a yaw measurement every cycle. A 120°/cycle spike filter rejects corrupt readings. Yaw state is corrected via a standard Kalman update.

10 Hz — GPS Update

Position Correction

On each new GPS fix, the x/y position state is corrected. GPS course is blended into the yaw estimate when speed exceeds 0.8 m/s. Velocity is zeroed when the car is stationary.

Output

UTM pos/x · pos/y

Absolute UTM easting and northing fed to DonkeyCar's path follower at 50 Hz, the same coordinate system used when recording waypoints, so no other changes are needed.
