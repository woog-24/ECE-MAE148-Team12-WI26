03 — FINAL PROJECT

GPS + IMU Sensor Fusion

Comparing a budget GPS + external IMU (with Kalman filtering) against a premium all-in-one $300 GPS/IMU unit for autonomous track navigation.

💸 Budget Setup
  • Cheap GPS module
  • External IMU sensor
  • Custom Kalman filter
  • Manual sensor fusion code
  • Cost-effective solution
💰 Premium Setup
  • ~$300 GPS unit
  • Internal IMU (onboard)
  • Onboard sensor fusion
  • Plug-and-play integration
  • High accuracy out-of-box

PROJECT OBJECTIVES

Our primary goal was to evaluate whether a low-cost GPS + external IMU combination, enhanced by a Kalman filter, can match or approach the navigation accuracy of an expensive integrated GPS/IMU solution on an autonomous track.

  • Integrate a budget GPS module and external IMU with the DonkeyCar framework
  • Implement a Kalman filter for GPS + IMU sensor fusion
  • Complete autonomous laps using the budget sensor stack
  • Benchmark performance against the $300 premium GPS/IMU unit
  • Document the accuracy, latency, and reliability differences between both setups

INTEGRATION OVERVIEW

We wrote Python code to bridge the GPS and IMU data streams into the DonkeyCar parts system. Below is a simplified snippet showing how sensor data is fused using a Kalman filter and passed to the vehicle controller.

PYTHON 
# GPS + IMU Kalman Filter Integration for DonkeyCar
# Team 12 — ECE/MAE 148 WI26

import numpy as np
from donkeycar.parts.gps import GPS
from imu_driver import IMU

class KalmanGPSIMU:
    def __init__(self):
        # State: [x, y, vx, vy]
        self.x = np.zeros(4)
        self.P = np.eye(4) * 1000
        self.Q = np.eye(4) * 0.1   # Process noise
        self.R = np.eye(2) * 5.0    # GPS measurement noise

    def predict(self, ax, ay, dt):
        # Predict step using IMU acceleration
        F = np.array([[1,0,dt,0],[0,1,0,dt],[0,0,1,0],[0,0,0,1]])
        B = np.array([[0.5*dt**2,0],[0,0.5*dt**2],[dt,0],[0,dt]])
        u = np.array([ax, ay])
        self.x = F @ self.x + B @ u
        self.P = F @ self.P @ F.T + self.Q

    def update(self, gps_x, gps_y):
        # Update step using GPS measurement
        H = np.array([[1,0,0,0],[0,1,0,0]])
        z = np.array([gps_x, gps_y])
        y = z - H @ self.x
        S = H @ self.P @ H.T + self.R
        K = self.P @ H.T @ np.linalg.inv(S)
        self.x = self.x + K @ y
        self.P = (np.eye(4) - K @ H) @ self.P
        return self.x[:2]  # Return fused position

  • GPS Signal Latency The budget GPS module had higher latency compared to the premium unit, causing the vehicle to react slowly to track changes. We partially mitigated this using predictive filtering from the IMU.
  • IMU Drift Accumulation Over longer laps, small errors in IMU readings accumulated (gyroscope drift), causing the Kalman filter's position estimate to diverge from the true path.
  • DonkeyCar Integration Complexity Connecting external IMU and GPS into DonkeyCar's parts pipeline required custom part classes and careful synchronization of data streams at different update rates.
  • VESC Tuning Getting consistent throttle and steering response from the VESC required manual PID tuning for our specific vehicle weight and wheel setup.

POTENTIAL FUTURE DIRECTIONS

If we were to continue this project, here are the improvements and explorations we would pursue:

  • Implement an Extended Kalman Filter (EKF) or Unscented Kalman Filter (UKF) for nonlinear motion models
  • Add RTK-GPS correction for centimeter-level positioning accuracy on the budget hardware
  • Fuse camera data with GPS+IMU for a robust multi-modal navigation system
  • Deploy the pipeline in ROS2 Nav2 for more sophisticated path planning
  • Test in GPS-denied environments and evaluate IMU-only dead reckoning performance
